# 代码块渲染方案重构

## 问题分析

当前方案经历了多轮补丁（`<pre>` -> `<section>+<p>` -> 背景色迁移），结构变得复杂且脆弱。核心矛盾是：**我们在用非语义化标签模拟 `<pre>` 的行为**。

实际上，微信公众号编辑器**确实支持** `<pre>` 的横向滚动——很多技术公众号（如 OSC 开源社区、前端之巅）的代码块就是可以横滑的 `<pre>` 标签。之前失败的原因不是 `<pre>` 本身，而是我们在 `<pre>` 上叠加了太多微信不认的 CSS 属性，以及用了 `<p>` / `<br>` 等块级元素破坏了 `<pre>` 的原生空白保留行为。

## 新方案：回归 `<pre>` 原生行为

**核心原则**：`<pre>` 标签天生就保留空白和换行，不需要 `&nbsp;` 和 `<br>`，也不需要 `white-space` 属性。微信也遵守这个语义。我们只需要：

1. 使用**原生 `<pre>`** 标签（不加 `white-space` 属性，让浏览器和微信都用默认行为）
2. 代码内容用**纯文本换行**（`\n`），不用 `<br>`
3. 空格用**原生空格**，不用 `&nbsp;`（`<pre>` 内的空格天然不折叠）
4. 着色仍用 `<font color>`（微信安全）
5. 外层用 `<section>` 包裹提供圆角和语言标签

```
<section border-radius:8px; overflow:hidden>
  <section>TypeScript</section>
  <pre background:#282c34; overflow-x:auto; font-size:13px>
    <code>
      <font color="#C678DD">const</font><font color="#E5C07B"> x</font> = 1
    </code>
  </pre>
</section>
```

关键区别：
- `<pre>` 内原生空格不折叠，不需要 `&nbsp;`
- `<pre>` 内原生换行有效，不需要 `<br>`
- `<pre>` 自带 `white-space: pre`，不需要额外设置
- `overflow-x: auto` 加在 `<pre>` 上实现横向滚动（微信支持 `<pre>` 的 overflow）
- `escapeHtml` 只做 HTML 转义（`<>&"`），不替换空格

## 改动文件

### [lib/markdown.ts](lib/markdown.ts)

- `escapeHtmlForCode()` 函数：移除 `&nbsp;` 替换，只保留标准 HTML 转义
- `buildWeChatCodeHtml()` 函数：
  - 回归 `<pre><code>` 结构
  - 行间用 `\n` 分隔，不用 `<br>`
  - `<pre>` 只设最少属性：`background-color`, `color`, `padding`, `margin`, `font-size`, `line-height`, `font-family`, `overflow-x:auto`
  - 不设 `white-space`（让 `<pre>` 用默认值）
  - 外层仍用 `<section>` 包裹（圆角 + 语言标签）
